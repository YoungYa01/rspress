# 毕设需求文档

## 题内容介绍

各级部门都需要收集和审查各类 资料（包括纸质或电子资料），如今资料的提交与审核反馈信息化程度不高，审查过程也难以留痕，本课题拟建一个通用的资料提交系统，收集资料的部门可以设置提交资料的人员，发布资料清单和要求，可以设置审核人员；提交资料者可以提交资料和查看资料审核进程等；审核人员能审核，管理员可以查看和导出审查记录等。

## 用户

- 超级管理员
- 教师
- 学生

## 文件类型

1. word
2. excel
3. pdf
4. txt
5. zip
6. png、jpg······

## 超级管理员权限

- [x] All of the permissions
- [x] 查看所有页面
- [x] 管理用户、角色、菜单、国际化、日志
- [ ] 导入用户

## 教师权限

- [x] 发布通知、公告、活动、消息
- [ ] 发布文件收集任务
- [ ] 添加文件收集的额外信息
- [ ] 管理某次收集任务的所有文件
- [ ] 打包导出文件
- [ ] 管理某次收集任务的有哪些人提交
- [ ] 文件预览：word、excel、pdf···等
- [ ] 管理纸质文件的存档信息

## 学生权限

- [x] 查看通知、公告、活动、消息
- [x] 查看管理个人信息
- [ ] 选择任务进行文件提交

## 附加功能

- [ ] 大文件上传
- [ ] 图片内容文字扫描
